## 3. The four arms — exact definitions

| Arm | ID | What is computed | Cell | Model(s) | Correction | Target field |
|-----|----|------------------|------|----------|------------|--------------|
| A | `raw-1x1x1` | Single-model elastic constants, conventional cell | 1×1×1 | TensorNet/PBE | None | `TPBE_0K` |
| B | `corrected-1x1x1` | Same as A + Lupine operator (LOO bias-PCA + functional shift) | 1×1×1 | TensorNet/PBE | LOO-PCA bias + shift | `TPBE_0K` |
| C | `ref-3x3x3` | Single-model elastic constants, 3×3×3 supercell | 3×3×3 | TensorNet/PBE | None | `TPBE_0K` |
| D | `ensemble-1x1x1` | Mean of 3 architectures, conventional cell | 1×1×1 | M3GNet+PBE, CHGNet+PBE, TensorNet+PBE | None | `TPBE_0K` |

**MLIP elastic benchmark pairwise comparisons:**
- B vs C → does corrected small cell match big-cell reference? (`MAE(B) ≤ MAE(C) + ε`, ε = 1.0 GPa)
- B vs D → does corrected single model beat ensemble? (`MAE(B) < MAE(D)`)
- A vs B → sanity: does operator help? (`MAE(B) < MAE(A)`)
- cost(C)/cost(B) and cost(D)/cost(B) → the 10× / 5× multipliers

## 5. LOO bias-PCA procedure (arm B)

### 5.1 Algorithm

For each element `e` in the 16-element roster:

1. Collect the TensorNet/PBE 1×1×1 predictions for the **other 15 elements**:
   - `raw_i = [c11_i, c12_i, c44_i]` for i ≠ e
   - `target_i = TPBE_0K[i]`
   - `error_i = raw_i − target_i` (3-vector)

2. Center the 15 error vectors: `E = [error_1, ..., error_15]` (3×15 matrix)

3. Compute the first principal component of `E` (SVD, `np.linalg.svd`):
   - `b_e = U[:,0] * S[0]` — the 3D bias vector for element `e`

4. Compute the functional shift for element `e`:
   - `shift_e = Tr2SCAN_0K[e] − TPBE_0K[e]` (from `targets_0K.json`)
   - **Headline only:** for PBE headline, `shift_e = 0` (same functional)
   - **Sensitivity:** apply `shift_e` for r2SCAN comparison

5. Apply correction:
   - `corrected_e = raw_e − b_e + shift_e`

6. Compute MAE for element `e`:
   - `mae_e = mean(|corrected_e − TPBE_0K[e]|)`

7. Aggregate:
   - `MAE(B) = mean(mae_e over 16 LOO folds)`
   - Report both **mean** and **median** (Cr is a known outlier at ~43 GPa)

### 5.2 Implementation

The operator module (`lupine/python/lupine/operator.py`) already provides:
- `leave_one_out_calibration(rows)` — returns conformal residuals
- `compare_operator_to_ensemble(...)` — head-to-head scorecard

For the mlip elastic benchmark we need a thin wrapper that:
1. Reads the 16 per-case JSONs (or the benchmark aggregation JSON)
2. Builds the 15×3 error matrix per fold
3. Calls `np.linalg.svd` for the bias vector
4. Calls `correct(raw, bias, shift)` for each element
5. Writes the corrected predictions to `mlip_elastic_benchmark_results.json`

**Reference implementation location:** `lupine/python/lupine/mlip_elastic_benchmark_loo_bias.py` (to be created by execute task, but the algorithm is specified here).

### 5.3 Why LOO is mandatory

In-sample bias fitting (fitting on all 16 including `e`) would leak the target for `e` into its own bias vector. The operator would appear to win by construction. The leave-one-out protocol is the only honest estimator for a 16-element benchmark.

## 6. Cost model (cache-warm core-hours)

### 6.1 Currency

```
core_seconds = runtime_seconds × n_cores
core_hours   = core_seconds / 3600
```

For `matcalc.ElasticityCalc` runs:
- `n_cores = 1` (single-threaded relax + stress-strain)
- Host core count recorded separately for node-hours

### 6.2 Cache-warm requirement

All headline cost numbers MUST come from cache-warm runs (models pre-downloaded to `~/.cache/matgl/` or `~/.cache/huggingface/`). The existing 1×1×1 total of 1217.8 s is cache-cold and slower than 3×3×3 (519.5 s) — physically impossible, purely download overhead.

**Warm-cache pre-step (execute task must run this first):**

```bash
# Pre-download all three models so subsequent runs are warm
python -c "from matcalc import load_up; [load_up(m) for m in [
    'M3GNet-PES-MatPES-PBE-2025.2',
    'CHGNet-PES-MatPES-PBE-2025.2.10',
    'TensorNet-PES-MatPES-PBE-2025.2'
]]"
```

Discard any runtime > 60 s for a 1×1×1 case as cache-cold outlier.

### 6.3 Cost per arm

| Arm | Runtime source | Formula | Expected scale |
|-----|---------------|---------|----------------|
| A raw-1x1x1 | Cache-warm 1×1×1 runs | `sum(runtime_seconds) / 3600` | ~0.3–0.5 core-hours (16 elem) |
| B corrected-1x1x1 | = A + LOO overhead | `cost(A) + negligible` | ~0.3–0.5 core-hours |
| C ref-3x3x3 | Existing `layer2_outputs_3x3x3_16elem/` | `sum(runtime_seconds) / 3600` | ~0.5–1.0 core-hours (16 elem) |
| D ensemble-1x1x1 | = 3 × A (M3GNet + CHGNet + TensorNet) | `3 × cost(A)` | ~0.9–1.5 core-hours |

**Note:** The existing 3×3×3_16elem data shows mean runtime ~10.87 s per case (112 cases). For 16 elements × 1 model = 16 cases, expect ~170 s total. The Cu/Ni 3×3×3 data shows ~32.47 s mean. Use the actual measured runtimes from the existing data, not estimates.

## 7. Exact CLI invocation matrix

### 7.1 Warm-cache pre-step

```bash
python -c "from matcalc import load_up; [load_up(m) for m in [
    'M3GNet-PES-MatPES-PBE-2025.2',
    'CHGNet-PES-MatPES-PBE-2025.2.10',
    'TensorNet-PES-MatPES-PBE-2025.2'
]]"
```

### 7.2 Arm A: raw-1x1x1 (TensorNet/PBE, all 16 elements)

```bash
for elem in Ag Al Au Ca Cr Cu Fe Mo Nb Ni Pd Pt Sr Ta V W; do
  python lupine/data/layer2_benchmark_task.py \
    --element $elem \
    --model TensorNet \
    --functional PBE \
    --supercell 1 \
    --output "results/armA_raw_1x1x1/${elem}_TensorNet_PBE.json"
done
```

**Cu and Ni already exist** in `lupine/data/layer2_outputs/` — copy, do not re-run.

### 7.3 Arm C: ref-3x3x3 (TensorNet/PBE, all 16 elements)

**Already exists** in `lupine/data/layer2_outputs_3x3x3_16elem/`.  
Copy to `results/armC_ref_3x3x3/` for aggregation. Do not re-run.

### 7.4 Arm D: ensemble-1x1x1 (M3GNet + CHGNet + TensorNet, all 16 elements)

```bash
for elem in Ag Al Au Ca Cr Cu Fe Mo Nb Ni Pd Pt Sr Ta V W; do
  for model in M3GNet CHGNet; do
    python lupine/data/layer2_benchmark_task.py \
      --element $elem \
      --model $model \
      --functional PBE \
      --supercell 1 \
      --output "results/armD_ensemble_1x1x1/${elem}_${model}_PBE.json"
  done
  # TensorNet already in armA — copy
  cp "results/armA_raw_1x1x1/${elem}_TensorNet_PBE.json" \
     "results/armD_ensemble_1x1x1/${elem}_TensorNet_PBE.json"
done
```

### 7.5 Arm B: corrected-1x1x1 (post-hoc, no new matcalc runs)

No CLI invocations. The execute task runs `mlip_elastic_benchmark_loo_bias.py` (§5.2) which:
1. Reads arm A results
2. Reads `targets_0K.json`
3. Computes LOO bias vectors
4. Applies `correct(raw, bias, shift)`
5. Writes corrected results to `results/armB_corrected_1x1x1/`

### 8.2 Validation rules

- `schema_version` MUST be exactly `"lupine.mlip_elastic_benchmark.v1"`
- `arms.corrected-1x1x1.bias_method` MUST be `"LOO-PCA"`
- `arms.ensemble-1x1x1.n_models` MUST be 3 (not 4)
- `per_element` MUST contain exactly 16 rows per arm (64 total)
- `cost_ratios.corrected_vs_ref` MUST be ≥ 1.0 (corrected should be cheaper)
- All `runtime_seconds` in per-element MUST be < 60 s for 1×1×1 cases (cache-warm check)

## 10. Risk register (pre-planned responses)

| Risk | Trigger | Response |
|------|---------|----------|
| Operator does NOT beat ensemble on MLIPs | `MAE(B) > MAE(D)` | Report honestly. Pivot headline to supercell-independence cost saving (B vs C). The 10× claim survives; the 5× ensemble claim does not. |
| Cache-cold timings contaminate cost | Any 1×1×1 runtime > 60 s | Discard outlier, re-run with warm cache, document in `provenance.cache_warm` |
| Model download failure | `matcalc.load_up()` raises | Retry once, then fall back to cached checkpoint if available; document in provenance |
| Cr outlier skews mean | `MAE(Cr) > 40 GPa` | Expected. Report median alongside mean; median is the defensible headline. |
| LOO fold produces non-physical bias | `np.linalg.svd` returns NaN | Check for zero-variance error matrix (all errors identical). If so, bias = mean error vector. Document. |
