# Partnerships

Lupine's external alignment, under a deliberate visibility convention:

- **Public:** the scientific thesis, the methodology, and which problem classes the work
  maps onto.
- **Gated:** commercial terms, named counterparties where not already public, and
  unpublished negative results tied to a specific engagement.

This page is the public layer.

## The thesis being validated

Revealed preference in the field is that funding flows to **data-driven, closed-loop
potential validation and retraining**, not to incrementally better DFT. Lupine's bet —
the audit layer for the MLIP ecosystem, compounding into a low-rank retraining target —
is paradigm-aligned with where the money and the open problems actually are.

## MIIT-67 pilot mapping

The MIIT-67 pilot programs were mapped against the Lupine thesis. Tier-1 direct hits —
problem classes where the hyper-ribbon error-geometry method applies with little
adaptation:

| Pilot wedge | Why it maps |
|-------------|-------------|
| **#2 — Cu alloys** | Elastic-constant error geometry is exactly the corpus's strongest regime. |
| **#18 — Mg stiffness** | Stiffness prediction is a direct manifold-projection use case. |
| **#15 — Powder reuse** | Property-transfer validation across reuse cycles is a closed-loop fit. |

The corpus is metals-elastic-heavy today; the *methodology* transfers beyond that, and
the Phase-D compute lane extends property coverage ($E_\text{coh}$, $B_0$) to widen the
addressable set.

## Named pilot status

Two pilots have been instrumented and run through the live traced loop:

- **Cu (#2):** stable at `testing` confidence ~0.3 — an honest weak, non-confirming
  signal, reported as such.
- **Mg (#18):** surfaced a pooled-eval gap that the demo itself exposed and which was
  then fixed (Mg routed to the element pattern).

Next: Phase-D elastic-constant runs on Cu/Mg to move these from weak signal to grounded
result.
